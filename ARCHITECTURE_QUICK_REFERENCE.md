```
ADD A NEW EVENT
  1. Add to _events object in utils.ts
  2. Update EventMap type if using typed events
  3. Implement listener in platform entry point
  4. Dispatch from component

ADD A NEW UI COMPONENT
  1. Create in src/shared/
  2. Import in ws-flow-page.ts or ws-flow-list-page.ts
  3. Use m() to render
  4. Access state via services

ADD SERVICE METHOD
  1. Add method to FlowService or FlowListService
  2. Update state
  3. Call dispatchUpdated() if needed
  4. Use in components

STYLE A COMPONENT
  1. Use Tailwind utility classes
  2. Use DaisyUI component classes
  3. Override with style.css if needed
  4. Use data-theme for theming

DEBUG AN ISSUE
  1. Check browser console for errors
  2. Verify event is dispatched (Network tab)
  3. Check service state in debugger
  4. Verify host listener is registered
  5. Check Mithril redraw is triggered
```

| Aspect | Implementation |
|--------|---------------|
| **UI Framework** | Mithril.js (lightweight, 10kb) |
| **State Management** | Global services (FlowService, FlowListService) |
| **Communication** | Event-driven (CustomEvent API) |
| **Styling** | TailwindCSS + DaisyUI |
| **Type Safety** | TypeScript throughout |
| **Build System** | Rollup (multi-target bundling) |
| **Code Reuse** | 95% shared, 5% platform-specific |
| **Bundle Size** | ~100kb minified (per target) |
| **Browser Support** | Modern browsers (ES6+) |
| **Dependencies** | Minimal (4 core deps) |
